# AviUtl / AviUtl2 連携 実機テスト手順

V-Adapter の「音声・字幕 投げ込み連携」を実機で確認する手順です。
AviUtl（無印）優先で検証します。

---

## 0. 事前準備（前提環境）

以下がインストール・起動済みであること（既存の かんしくん/PSDToolKit 環境と同じ前提です）。

- **AviUtl 本体 + 拡張編集（ExEdit）**
- **ごちゃまぜドロップス（gcmzdrops）** … AviUtl の `plugins` に導入済み
- **PSDToolKit** … wav+txt から字幕・口パクを生成するドロップハンドラ
- AviUtl で**プロジェクト（新規でも可）を開いておく**
  - 解像度・フレームレートが設定された状態（gcmz はプロジェクト未読込だと投入不可）

> V-Adapter は wav と同名 txt を gcmz の外部連携APIへ「一緒に投げる」だけです。字幕・口パク等の
> オブジェクト生成は PSDToolKit 側のドロップ処理が担います。

### PSDToolKit 環境設定 > 発動条件（重要）
PSDToolKit の「\*.wav ファイルドロップ拡張」→「発動条件」で、必ず
**☑「同じ名前の \*.wav と \*.txt を一緒にドロップした時」** を有効にしてください。

- V-Adapter は wav と txt を**一緒に**投げるため、この条件で口パク準備・字幕準備等が生成されます。
- 「Shift を押しながら…」は API ドロップでは Shift 状態を渡せないため発動しません。
- 「\*.exo をドロップした時」は V-Adapter が .exo を作らないため対象外です。
- 「字幕準備」の **文字コード** は txt の実際のエンコーディングに合わせます（字幕が文字化けする場合は
  UTF-8 / Shift_JIS を切り替え）。

### テスト用の wav+txt ペアを用意
適当なフォルダ（例 `C:\test\voice`）に、**同じ名前**の wav と txt を置きます。
```
C:\test\voice\0001_ずんだもん_こんにちは.wav
C:\test\voice\0001_ずんだもん_こんにちは.txt   ← 中身は字幕テキスト「こんにちは」
```
（合成音声ソフトの実際の書き出しを使ってもOK）

## 1. V-Adapter を起動して連携モードにする

1. `V-Adapter.exe`（または `dotnet run --project src/VAdapter.App`）を起動。
2. ツールバーの **「連携設定」** を開く。
3. 連携環境で **「AviUtl + PSDToolKit」** を選択。

---

## 2. gcmz 接続確認（状態表示）

連携設定の上部「状態」を確認し、**「状態更新」** を押す。

| 表示 | 意味 |
|------|------|
| `ごちゃまぜドロップス: 接続OK` / `プロジェクト: 読込済み（1920x1080 / API v…）` | ✅ 正常 |
| `ごちゃまぜドロップス: 未検出` | AviUtl かプラグイン未起動 → 起動を確認 |
| `プロジェクト: 未読込` | AviUtl でプロジェクトを開く |

> ✅ ここで「接続OK」「読込済み」が出れば、外部連携APIの疎通は成立しています。

---

## 3. テスト投げ込み（最小の疎通確認）

1. 連携設定の **「今すぐ投げる（テスト）…」** を押す。
2. 用意した `0001_ずんだもん_こんにちは.wav` を選択。
3. **AviUtl のタイムラインにオブジェクトが追加**されることを確認。
   - PSDToolKit が設定済みなら、**音声＋字幕**が「既定レイヤー」（初期値=レイヤー1）付近に載る。
4. 連携設定のログに `テスト投入成功: …` が出る。

> 失敗時はログに理由（未検出／未読込 等）が表示されます → §7 参照。

---

## 4. フォルダ監視の自動投入

1. 連携設定の **「監視フォルダ」→「追加…」** で `C:\test\voice` を登録（サブフォルダも監視したい場合はチェック）。
2. **「OK」** で設定を保存（監視が開始します。メイン画面の実行ログに `[連携] 監視開始: …` が出る）。
3. 監視フォルダへ **新しい同名 wav+txt ペアをコピー**（または合成音声ソフトで書き出し）。
   - 例: `0002_四国めたん_テストです.wav` ＋ `.txt`
4. 数百ミリ秒〜数秒で**自動的にタイムラインへ投入**され、メイン画面ログに
   `[連携] 検知: …` → `[連携] 投入成功: …` が出ることを確認。

確認ポイント：
- 書き込み途中で**二重投入されない**こと（安定待ち後に1回だけ投入）。
- 同名 txt が無い wav は投入されない（ログに「.txt が見つからず無視」）。

---

## 5. 合成音声ソフトの保存マクロとの連携（end-to-end）

1. 合成音声ソフトの**書き出し先フォルダ**を §4 の監視フォルダに設定（またはそのフォルダを監視に追加）。
2. V-Adapter の **「音声の保存」マクロ**（Ctrl+E 等）で wav+txt を書き出す。
3. 書き出し完了 → 監視が検知 → AviUtl へ自動投入、までが一気通貫で動くことを確認。

---

## 6. 話者ルール（レイヤー振り分け）

1. 連携設定の **「話者ルール」→「行を追加」** で、ファイル名パターン（正規表現）・話者名・レイヤーを設定。
   - 例: パターン `ずんだもん` → レイヤー `5` / パターン `四国めたん` → レイヤー `7`
   - パターンにキャプチャグループ（例 `^\d+_(.+?)_`）を使うと話者名を自動抽出。
2. 「OK」で保存し、§3〜§4 を再実行。
3. ファイル名に応じて**意図したレイヤー**へ載ること、未一致は**既定レイヤー**に載ることを確認。

---

## 7. トラブルシューティング

| 症状 | 確認 |
|------|------|
| 「未検出」のまま | AviUtl と ごちゃまぜドロップスの起動／プラグイン導入。別の AviUtl が起動していないか |
| 「未読込」 | AviUtl でプロジェクトを開く（解像度設定） |
| 投入されるが字幕が出ない | PSDToolKit のドロップ設定。wav と**同フォルダに同名 txt** があるか |
| 監視が反応しない | 監視フォルダのパス。OK で設定保存したか（保存時に監視開始）。書き込み先が監視フォルダか |
| 文字化け／投入失敗 | パスに ShiftJIS で表現できない文字（絵文字等）が含まれていないか |

---

## 8. ログ・設定ファイルの場所

- 連携ログ: メイン画面「実行ログ」（`[連携] …`）、および連携設定画面のログ欄。
- 設定ファイル: `%APPDATA%\V-Adapter\integration.json`（モード・監視フォルダ・話者ルール）。

---

## 9. AviUtl2（後続検証）

外部連携APIはマッピング名・ミューテックスが共通のため、連携環境で
**「AviUtl2 + PSDToolKit2」** を選ぶと dwData=2（＋margin）経路で投入します。
GCMZDrops2 / PSDToolKit2 環境で §2〜§6 を同様に確認してください
（`margin` 欄が AviUtl2 のとき表示されます）。
